## Getting Started

1. Fork the repository and create a new branch from `main`.
2. Keep changes focused: one bug fix or feature per pull request is ideal.
3. Write clear commit messages that explain *why* the change was made.
4. Update documentation or comments where appropriate.

## Reporting Issues

When opening an issue, please include:

- A clear and descriptive title
- What you expected to happen
- What actually happened
- Steps to reproduce (if applicable)
- Relevant environment details (OS, version, etc.)

Minimal reproducible examples are especially appreciated.

## Pull Requests

For pull requests:

- Reference any related issues (e.g., `Fixes #12`)
- Explain the motivation and approach in the PR description
- Keep pull requests reasonably small to make review easier
- Ensure existing tests pass (and add new tests if appropriate)
